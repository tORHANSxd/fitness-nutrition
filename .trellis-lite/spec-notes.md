# Spec Notes

- 公共食物编辑规则：网页端编辑公共食物时保存为当前用户的覆盖值，食物 id 保持 public-* 不变，计划计算直接使用覆盖后的营养值；重置只删除该用户覆盖值，不改内置公共库和其他用户数据。Supabase 需要 public.food_overrides 表和 own-user RLS。
- 默认分类克重/上限：熟主食 180g/360g，生主食 60g/120g，蔬菜 200g/420g，水果 120g/250g，肉类 150g/260g，坚果 20g/35g，乳清/酪蛋白类补剂 30g/40g，肌酸 5g/5g，鱼油 2g/5g，电解质 8g/20g；用户显式 maxGrams 优先于默认上限。
- 求解约束优先级：锁定整餐和锁定食物不改动；用户 minGrams/maxGrams 为硬约束；分类默认上限用于未设置 maxGrams 的食物；份量软约束只用于减少极端推荐，不强制替代宏量营养素目标。
- 全天推荐贴合规则：展示用每餐目标只按每日目标和餐次比例计算，不能被求解器改写；求解器先按每餐目标生成初始推荐，再按全天目标对可调整食物克重做二次优化；整餐锁定和锁定食物保持不变。
- 全天求解评分规则：推荐克重的最终优化目标是让全天热量、碳水、蛋白、脂肪四项绝对差距同时变小；盈余和亏空都允许，评分只看距离日目标有多远，蛋白权重略高用于避免碳水/热量补足时过量堆高蛋白；份量回拉和餐内结构约束用于打断极端单食物解。
- 餐内多样性求解规则：多食材同餐时，求解器需要给已加入食物保留类别存在下限，并给显式大 maxGrams 套一层现实硬上限；主食现实上限为默认份量 * 3，蔬菜/水果为 * 2.1，肉类为 * 1.8，补剂为 * 1，坚果为 * 1.5。评分同时惩罚单个非补剂食物克重占比过高和餐内类别重量偏离。
- 默认演示餐约束：默认计划不设置强制最小克重；高碳默认演示餐给主食/水果更高上限，便于自动推荐贴近日目标，用户可在网页端自行收紧任意食物的 min/max。
- 当日盈亏显示规则：热量、碳水、蛋白、脂肪都显示当前摄入与应用推荐后的全天总量；柱状比例 = 摄入量 / 目标量，盈亏 = 目标量 - 摄入量。

- BMR = 基础代谢率，使用 Mifflin-St Jeor：男性 10*体重kg + 6.25*身高cm - 5*年龄 + 5；女性 10*体重kg + 6.25*身高cm - 5*年龄 - 161。
- TDEE = 维持热量 = BMR * 日常活动系数 + 当日运动消耗kcal；日常活动系数按不含当天训练消耗填写，避免训练消耗重复计入。
- 目标热量 = TDEE ± 体重变化热量；减脂默认每周 -0.5% 体重，允许 0.25%-1.0%；增肌默认每周 +0.25% 体重，允许 0.1%-0.5%；换算采用 7700 kcal/kg，减脂单日缺口上限 min(1000 kcal, TDEE*25%)，增肌单日盈余上限 min(500 kcal, TDEE*15%)。
- 食物分类固定为：主食、蔬菜、水果、肉类、补剂、坚果。
- 食物营养字段固定为每 100g 的热量、脂肪、碳水、蛋白质；重量口径为 raw/cooked，未知生熟换算时不显示另一口径。
- 用户输入克重时该食物行临时锁定，自动求解只给出推荐值；点击应用推荐后才改写未锁定食物克重。
- 三大营养素比例 = 各宏量营养素供能 / 碳蛋脂总供能；餐次盈亏 = 餐次目标 - 当前实际摄入，正数显示为亏，负数显示为盈。
